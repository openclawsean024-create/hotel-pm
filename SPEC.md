## 1. 問題陳述

### 1.1 目標使用者

| 族群 | 規模 | 痛點 |
|---|---|---|
| 民宿經營者（1-10 房） | ~5,000 | 訂房來自 Airbnb/Booking/官網，需手動彙整；房客資料分散 |
| 包租代管業者（管理 50-200 房） | ~800 | 多物件 + 多房客 + 多平台訂單，紙本表單無法應付 |
| 短租日租套房經營 | ~3,000 | 每日入住退房、清潔排程、設備維修追蹤混亂 |

### 1.2 為什麼不做替代方案

- **Excel 自製表**：多物件多房客時易出錯、無提醒、協作困難
- **商用 PMS（Cloudbeds/Hostaway）**：月費 50-200 USD，對台灣微型民宿太貴
- **紙本記事本**：無法搜尋、易遺失、無法多人協作
- **我們的解法**：純前端 SPA + localStorage + 6 大模組（房客/物業/訂房/維修/月報/分潤），零月費

---

## 2. 解決方案

### 2.1 核心價值主張

> 「6 個分頁管好 200 間房 — 房客 CRM、物業管理、訂房看板、需求記錄、維修派工、月報表全包。」

### 2.2 使用者流程

1. 新增物業（地址、房型、坪數、月租金、押金）
2. 新增房客（姓名、電話、合約期間、租金付款日）
3. 建立訂房記錄（入住日/退房日/平台來源 Airbnb/Booking）
4. 房客報修時，新增維修單（描述、照片、廠商指派）
5. 月底一鍵產出月報表（房東分潤自動拆帳）

---

## 3. 功能清單

### 3.1 MVP（必做）

- [ ] 物業管理（地址/房型/坪數/照片/狀態 空置/出租中/維修中）
- [ ] 房客 CRM（姓名/電話/合約/租金/緊急聯絡人）
- [ ] 訂房看板（行事曆檢視 + 列表檢視，標示入住/退房/待清潔）
- [ ] 需求記錄（房客報修、特殊需求、清潔備註）
- [ ] 維修派工（單據 CRUD、指派廠商、完工日、費用）
- [ ] 月報表 + 房東分潤自動拆帳

### 3.2 v2（加值）

- [ ] 多平台訂單匯入（Airbnb/Booking ICS 檔）
- [ ] LINE 通知房客（合約到期、繳費提醒）
- [ ] 收據 PDF 自動產生
- [ ] 多用戶權限（房東/管家/清潔工）

### 3.3 明確不做

- 金流收款（轉帳/月結為主，不接金流 API）
- 稅務申報（交給會計師）
- 多國語系（先繁中）
- 與 Airbnb API 即時同步（v3 規劃）

## 6. 風險與決策

| 風險 | 等級 | 緩解 |
|---|---|---|
| localStorage 容量（200 房 + 1000 訂房） | 🟠 中 | 分頁儲存、只保留 12 個月熱資料、JSON 備份 |
| 個資疑慮（房客個資） | 🟠 中 | 全 localStorage、不上傳雲端、隱私頁說明 |
| 多裝置協作需求 | 🟡 低 | v2 加雲端同步，先做單機版 |
| 房東分潤錯誤（金錢問題） | 🔴 高 | 雙重驗算邏輯 + 警告提示 + JSON 匯出供會計核對 |
